# Template de Relatorio

Use este formato na Fase 2.

```markdown
# Relatorio de Auditoria Arquitetural

## Projeto

- Nome: ...
- Stack: ...
- Dominio: ...
- Arquivos analisados: ...
- Linhas aproximadas: ...
- Arquitetura atual: ...

## Sumario

- CRITICAL: ...
- HIGH: ...
- MEDIUM: ...
- LOW: ...
- Total: ...

## Findings

### [SEVERITY] Titulo do achado

- Arquivo: `path/to/file.ext:linha` ou `path/to/file.ext:inicio-fim`
- Categoria: ...
- Descricao: ...
- Impacto: ...
- Evidencia: ...
- Recomendacao: ...

## Riscos da Refatoracao

- ...

## Plano de Refatoracao Proposto

1. ...
2. ...
3. ...

## Criterios de Validacao

- [ ] Aplicacao inicia sem erros.
- [ ] Endpoints originais respondem.
- [ ] Estrutura MVC criada.
- [ ] Configuracoes sensiveis extraidas.
- [ ] Tratamento de erro centralizado.
```

Regras:

- Ordene findings por severidade: `CRITICAL`, `HIGH`, `MEDIUM`, `LOW`.
- Cada finding deve ter arquivo e linha exatos.
- Nao use achados genericos sem evidencia.
- Inclua no minimo 5 findings quando houver codigo suficiente.
