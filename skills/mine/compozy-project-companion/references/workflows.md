# Compozy Companion Workflows

## New Project Without Idea

Goal: help the user reach a clear Compozy-ready brief, not a PRD.

Required outputs before Compozy:

- chosen problem;
- target user;
- value proposition;
- MVP boundary;
- preferred stack constraints;
- AI/LLM/RAG involvement;
- external integrations;
- known risks;
- unresolved questions.

## Existing Codebase

Required outputs before Compozy:

- stack and architecture summary;
- run/test/build commands;
- major modules and entrypoints;
- documentation gaps;
- technical risks;
- suggested change boundary;
- what Compozy should receive as context.

## Post-Compozy Review

Check:

- artifact consistency;
- missing acceptance criteria;
- unclear dependencies;
- tasks too large;
- missing tests;
- missing operational constraints;
- LLM-specific risks when applicable.
